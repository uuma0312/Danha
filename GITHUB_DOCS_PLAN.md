# Danha GitHub docs plan

## Goal

Prepare the first public-facing documentation for Danha before publishing it to GitHub.

## Files

- `README.md`: introduce Danha, show the quickest way to run it, and explain the current project shape honestly.
- `docs/SYNTAX.md`: document the practical Danha syntax currently visible in examples and smoke tests.

## Tone

- Korean first, because the project and early users are Korean.
- Honest about the language being experimental.
- Focus on runnable examples instead of marketing claims.
